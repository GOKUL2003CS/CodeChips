HTML

Hyper Text Markup Language

HTML (Hypertext Markup Language) is the standard language used to create and structure web pages. It uses a series of tags and attributes to define the layout and content of a webpage, such as headings, paragraphs, images, and links. HTML is interpreted by web browsers and is the foundation upon which all websites are built. The latest version is HTML5 which introduced new elements and features such as multimedia support and semantic tags.

CSS

Cascading Style Sheet

CSS (Cascading Style Sheets) is a language used to control the layout and design of web pages. It allows you to separate the presentation of a document from its structure. CSS provides features such as selectors, properties, and values that can be used to change the appearance of specific elements on a webpage. CSS3 is the latest version, which introduced new features such as animations, transition, and responsive design.

JS

JavaScript

JavaScript is a programming language that is commonly used to create interactive and dynamic web pages. It is a client-side language, meaning it runs on the user's computer rather than the server. JavaScript can be used for a variety of tasks such as form validation, creating animations, and manipulating the Document Object Model (DOM). It is also often used in combination with other languages such as HTML and CSS to build complete web applications.

HTML 5 Introduction

HTML (Hypertext Markup Language) is a standard markup language used to create web pages. It consists of a series of tags and attributes that define the structure and content of a webpage.

Here is an example of a basic HTML5 document structure:

< !DOCTYPE html >
< html >
< head >
< title > Code Chips < / title >
< / head >
< body >
< h1 > Page Heading < / h1 >
< p > Page Paragraph < / p >
< / body >
< / html >

Explanation

The basic structure of an HTML document consists of the following elements:

1.Doctype declaration: This is the first line of an HTML document and it tells the web browser which version of HTML is being used. For example, the doctype declaration for HTML5 is <!DOCTYPE html.>

2.HTML: The main container for an HTML document, which wraps all other elements. It has an opening <html tag and a closing </html> tag.

3.head: This contains information about the document that is not displayed on the web page, such as the title of the page, meta data, and links to CSS and JavaScript files. It has an opening <head> tag and a closing </head> tag.

4.Title: This is the title of the web page, which is displayed in the browser's title bar or tab and is used by search engines to understand the content of the web page. The title is placed within the <head> section and is wrapped in <title> tags.

5.Body: This contains the content that is displayed on the web page. It has an opening <body> tag and a closing </body> tag.

HTML Paragraphs

In HTML, a paragraph is defined using the <p> element. The text within the <p> tags will be displayed as a block-level element (i.e. it will take up the full width of its parent container and create a new line before and after the element). For example:

< p > This is a Paragraph. < /p >
< p > This is another Paragraph. < /p >

A paragraph always starts on a new line, and is usually a block of text.

A paragraph always starts on a new line, and browsers automatically add some white space (a margin) before and after a paragraph.

HTML Horizontal Rules

The hr (horizontal rule) tag in HTML is used to create a horizontal line across the width of the parent container, separating different sections of content. The hr tag is a self-closing tag, which means that it doesn't have a closing tag.

Example:

< p > This is another Paragraph. < /p >

< p > This is another Paragraph. < /p >

HTML Line Breaks

In HTML, the br (line break) tag is used to create a line break, which is used to separate text into multiple lines. The br tag is a self-closing tag, which means that it doesn't have a closing tag.

< p > This is the first line of Paragraph. < br > This is the Second line of the Paragraph. < /p >

This will create two lines of text within the p tag, with a line break between them. The br tag does not have any attributes, but you can use CSS to control the spacing between lines, such as line-height.

HTML Preformatted Text

In HTML, the pre (preformatted text) tag is used to create preformatted text, which is text that is displayed in a fixed-width font and preserves both spaces and line breaks. The pre tag is used to enclose the preformatted text.

For example:

At the time of death it doesn’t stay back even if ordered;
If Goddess of wealth wills, wealth will multiply on its own
When she leaves, things will vanish
 < /pre >

This will display the text in a fixed-width font, preserving the spaces and line breaks as entered.

}

}

HTML Block & Inline Elements

HTML elements can be divided into two categories: block-level elements and inline elements.

Block-level elements create a block formatting context and take up the full width of their parent container. Examples of block-level elements include headings (h1, h2, etc.), paragraphs (p), and divs. These elements start on a new line and take up the full width of their parent container.

A block element can contain inline element.

Inline elements, on the other hand, only take up as much width as necessary and do not force a new line. Examples of inline elements include spans, anchors (a), and images (img). These elements can be used within a block-level element and will only take up as much width as the content inside them.

An inline element cannot contain a block-level element.

The HTML <code> element is used to define a piece of computer code. The content inside is displayed in the browser's default monospace font.

< code >
x = 5;
y = 6;
z = x + y;
< / code >

HTML <kbd> Element

The HTML <kbd> element is used to define keyboard input. The content inside is displayed in the browser's default monospace font.

< p > The following keyboard text are defined using kbd element. < kbd > Ctrl+S < / kbd > < / p >

HTML <samp> element

The HTML <samp> element is used to define sample output from a computer program. The content inside is displayed in the browser's default monospace font.

< p > Message from Computer: < samp > File not found.
Press F1 to continue. < / samp > < / p >

HTML <var> element

The HTML <var> element is used to define a variable in programming or in a mathematical expression. The content inside is typically displayed in italic.

< p > The area of a triangle is: 1/2 x< var > b < / var > x < var > h < / var >, where < var > b < / var > is the base, and < var > h < / var > is the vertical height.

HTML Entities

HTML entities are special characters or symbols that can be displayed in a web page.

By using HTML entities, you can display symbols or special characters that are not available on your keyboard, or you can display characters in a way that is not interpreted as HTML markup.

Some characters are reserved in HTML.

If you use the less than (<) or greater than (>) signs in your text, the browser might mix them with tags.

Character entities are used to display reserved characters in HTML.

A character entity looks like this:

&entity_name;
Or
&#entity_number;

To display a less than sign (<) we must write: &lt; or & #60;

Browsers may not support all entity names, but the support for entity numbers is good.

A commonly used entity in HTML is the non-breaking space: & nbsp;

A non-breaking space is a space that will not break into a new line.

Two words separated by a non-breaking space will stick together (not break into a new line). This is handy when breaking the words might be disruptive.

Another common use of the non-breaking space is to prevent browsers from truncating spaces in HTML pages.

If you write 10 spaces in your text, the browser will remove 9 of them. To add real spaces to your text, you can use the & nbsp; character entity.

Symbols that are not present on your keyboard can also be added by using entities.

HTML Forms

An HTML form is used to collect user input and submit it to a server for processing. Some common form elements include:

input: A form control where the user can input data. There are different types of inputs, including text, password, checkbox, radio, file, etc.

textarea: A form control for multi-line text input.

select: A form control for a drop-down list.

option: An option within a select form control.

button: A form control for a button that can be clicked to submit a form or perform other actions.

label: A form control for providing a label for an input element.

fieldset: A form control for grouping related form elements together.

legend: A form control for providing a caption for a fieldset.

form: A container for form elements such as: text fields, checkboxes, radio buttons, submit buttons, etc.

datalist: The <datalist> element specifies a list of pre-defined options for an <input> element.

Users will see a drop-down list of the pre-defined options as they input data.

The list attribute of the <input> element, must refer to the id attribute of the <datalist> element.

HTML input

The HTML input element is used to collect user input in a form. There are various types of input elements, including:

text: Allows the user to enter a single line of text.

password: Allows the user to enter a password, with the input displayed as asterisks.

checkbox: Allows the user to select one or more options from a list of choices.

radio: Allows the user to select one option from a list of choices.

submit: Allows the user to submit the form data to the server.

reset: Allows the user to reset the form data to its original values.

file: Allows the user to select a file from their computer to upload to the server.

hidden: Allows the user to include hidden data in the form that will be sent to the server.

image: Allows the user to submit the form data by clicking on an image.

date: Allows the user to select a date.

email: Allows the user to enter an email address.

number: Allows the user to enter a number.

action Attribute

The action attribute in an HTML form specifies the URL of the server-side script that will process the form data. When a user submits a form, the data is sent to the specified URL using the HTTP method specified in the method attribute (usually POST).

The input element is usually accompanied by a label element to provide a descriptive text for the input.

target Attribute

The target attribute specifies where to display the response that is received after submitting the form.

method Attribute

The method attribute specifies the HTTP method to be used when submitting the form data.

The default HTTP method when submitting form data is GET.

On GET method

Appends the form data to the URL, in name/value pairs. NEVER use GET to send sensitive data! (the submitted form data is visible in the URL!). The length of a URL is limited (2048 characters). Useful for form submissions where a user wants to bookmark the result.GET is good for non-secure data, like query strings in Google

On POST method

Appends the form data inside the body of the HTTP request (the submitted form data is not shown in the URL). POST has no size limitations, and can be used to send large amounts of data. Form submissions with POST cannot be bookmarked

autocomplete Attribute

The autocomplete attribute specifies whether a form should have autocomplete on or off.

When autocomplete is on, the browser automatically complete values based on values that the user has entered before.

Novalidate Attribute

The novalidate attribute is a boolean attribute.

When present, it specifies that the form-data (input) should not be validated when submitted.

HTML Input Attributes

HTML input elements can be restricted in several ways to ensure that the user enters valid data:

required

Makes an input element required and the form cannot be submitted without a value in this field.

pattern

Allows you to specify a regular expression pattern that the input value must match to be considered valid.

min and max

Restrict the minimum and maximum value allowed for an input element, respectively.

minlength and maxlength

specify the minimum and maximum number of characters allowed in an input field.

step

Restricts the increments in which a numerical value can be entered.

type

attribute: allows you to specify the type of input, such as email, number, date, etc., which sets additional restrictions and rules for the input value.

checked

Specifies that an input field should be pre-selected when the page loads (for type="checkbox" or type="radio")

disabled

Specifies that an input field should be disabled

readonly

Specifies that an input field is read only (cannot be changed)

size

Specifies the width (in characters) of an input field.

value

Specifies the default value for an input field.

multiple

The input multiple attribute specifies that the user is allowed to enter more than one value in an input field.

The multiple attribute works with the following input types: email, and file.

placeholder

The input placeholder attribute specifies a short hint that describes the expected value of an input field (a sample value or a short description of the expected format).

The placeholder attribute works with the following input types: text, search, url, tel, email, and password.

autofocus

The input autofocus attribute specifies that an input field should automatically get focus when the page loads.

list Attribute

The input list attribute refers to a <datalist> element that contains pre-defined options for an <input> element.

< form >
< input list= "browsers" >
< datalist id= "browsers" >
< option value= "Internet Explorer" >
< option value= "Firefox" >
< option value= "Chrome" >
< option value= "Opera" >
< option value= "Safari" >
< / datalist >
< / form >

autocomplete

The input autocomplete attribute specifies whether a form or an input field should have autocomplete on or off.

Autocomplete allows the browser to predict the value. When a user starts to type in a field, the browser should display options to fill in the field, based on earlier typed values.
